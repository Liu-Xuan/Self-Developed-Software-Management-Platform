INSTALLATION & BACKUP

THE AGS IV INSTALLATION

The AGS IV is shipped on a CD-ROM. This is the ground-based software that runs on a Personal Computer. It consists of the man-machine interface software, the database support tools, and the needed compilers.

 

During the process of building an airborne application (for loading on the target unit on the aircraft) the Target System Software (TSS) component, is also needed. This software, although also stored on the Personal Computer, executes only in the airborne unit. It is linked to the application during the build and loaded on the aircraft with the application.

 

Different TSSs exist for different aircraft types, such as Single Aisle and Long Range. Each TSS is also shipped on a separate CD-ROM. Only the TSSs applicable to the aircraft types under development need to be installed on the AGS IV PC. For example the TSS for Single Aisle aircraft need not be installed if only ACMS development for a Long Range aircraft is being done on the PC.

 

AGS IV and the TSS modules can be ordered from the Teledyne Controls Customer Administration Department. If the reader has access to a computer system with AGS IV, the needed TSS, and a sample application database already installed, then this chapter can be skipped in the first reading. However, this chapter should be read, when applications are being built or modified for production as it also covers the backup and restore functions; a necessary discipline for any computer based development.

 

The AGS IV build process (see Chapter 9 of this manual) also utilizes the Diab Data toolset.  This toolset is “node locked”, meaning a license file is required for the specific PC on which it is being used. The toolset software is included on the AGS IV installation CD, but a license file must be requested for each PC on which a build will be performed. This license file is needed only for the build process and database changes can be made without the license file.

The license file will be provided through Teledyne Controls.  In order to process this request, the disk serial number is required. The disk serial number may be read by typing “VOL C:” at the command line prompt in a DOS window.  (Be sure to get the serial number and not the label).  This information should be sent to Teledyne Controls Software Configuration Management, to the attention of Charles Gast.  The request may be sent via email to charles_gast@teledyne.com.

After Teledyne receives the license file text from Diab Data, it will be forwarded to the user via email.  A text-only file, named LICENSE.DAT, should then be created by the user in the folder C:\ISIPPC\LICENSES.  The license file text that you received via e-mail should be placed into the LICENSE.DAT file.

 

To install AGS IV on a PC:

(Note:  It is recommended to have, as a minimum, a Pentium II 450 MHz with 128 Megabytes of RAM and 2 Gigabytes of free space on your hard disk).

 

1.      insert the CD-ROM into the appropriate drive and follow the instructions on the screen. If the system does not start automatically,

2.      from the Windows  desktop, click the   button,

3.      select the Run option from the menu,

4.      when prompted for a filename, type d:\setup, (If  the CD-ROM on your computer is not the d: drive, use the appropriate drive letter instead),

5.      follow the instructions on the screen.

 

It is recommended to install AGS IV in either of  \Program Files\AGS IV or \AGS IV folders. If an older version of the program exists in the selected folder, the installation process will delete the older version.  Other than the folder selected, the installation also stores files in the following folders:  \MCC68K, and \ASM68K. The setup program will create all the necessary folders, if they do not already exist.

 

AGS IV Versions

As new functions are added to the AGS IV or a TSS, or the existing functions are refined, new versions of the installation disks are released. The different versions of such disks are identified by distinct letters of the alphabet. In most cases a newer version of a TSS can be installed without affecting the installed AGS IV or vice versa. In some cases, however, a new version of one component will also require an update to the other. AGS IV is aware of such requirements and if an attempt is made to open an application with an incompatible TSS, AGS IV will warn the developer but allow the development to continue.

 

This gives the developer an opportunity to order the new TSS or the AGS IV while development is continuing. However, attempting to do a build with a mismatched module will cause a build error, and thus prevent the upload of the application to the target.

APPLICATION INSTALLATION

Different applications, for different aircraft types, may be stored on a single PC. Each application should be given a unique name and saved in a separate folder, different from where the AGS IV is installed.

 

Once all the information necessary for an application is entered by the developer, AGS IV can generate the outputs that are to be uploaded to the aircraft. As a result of this step many more files and tables are automatically generated. These files and tables are stored in a subfolder inside a folder called AGS. The subfolder is given the same name as the application database. To have all files of an application in the same place, it is therefore recommended that the application database itself be also stored in the same subfolder.

 

For example the database file containing an application might be called 33TDY.mdb. The recommended place to save this file would be the folder c:\AGSIV\33TDY\ (i.e. the file will be referenced as c:\AGSIV\33TDY\33TDY.mdb). Thus, once AGS IV generates outputs for this application, all the related information are also placed in the c:\AGSIV\33TDY folder and its subfolders. 

 

Developed applications may be backed up and transported on removable media using either the Windows Backup or similar utilities from other sources (e.g Winzip). To install an application from a backed up media, the Restore utility of Windows (or the corresponding Unzip) should be used.  Backup (Winzip) and Restore (Unzip) may also be used for moving applications from one PC to another.

 

 

 

 

 

 

Application BACKUP AND RESTORE

As with any information saved on a hard disk, it is important to backup the ACMS applications in case the information on the hard disk is damaged or lost. This becomes even more important after an application is completed or updated and an uploadable production disk is generated for the fleet. In such cases the backed up version becomes the managed version of the application and subject to configuration control.

 

For all the above operations, the Backup utility of Windows may be used. Some Windows installations do not have the Backup/Restore utilities installed. In these cases the Windows installation CD-ROM may be needed to first install the needed utilities and then restore the application. 

 

Teledyne Controls provides a sample application that can be used to familiarize the customer with AGS IV. This application, as any other, is distributed in Windows format and must be restored with Windows Backup and Restore utility.

 

To install the Windows Backup utility, follow the supplier’s instructions. If the utility is already installed skip to the ‘Full and Source Backup’ section.

 

Full and Source Backup

The kind of information to be backed up from an application depends on the purpose of the backup. If the backup is to be used for archiving an application, then in addition to the developers inputs (i.e. the source), the compiled results and the upload image should also be backed up. This is called the ‘Full Backup’. On the other hand, if the purpose of a backup is to move an application from one computer to another, then only the developers’ inputs need to be backed up and the compilation steps may be repeated on the new machine. This limited backup is called a ‘Source Backup.
